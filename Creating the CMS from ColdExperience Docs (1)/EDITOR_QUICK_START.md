# Cold Experience CMS - Quick Start for Gustav

Welcome! This guide shows you how to edit content on the Cold Experience website.

## Accessing the CMS

1. Go to **https://admin.coldexperience.se**
2. Click **"Login with GitHub"**
3. Authorize the application
4. You're in! You'll see the CMS dashboard

## Main Sections You Can Edit

### 1. Hero Section
The big banner at the top of the homepage with the title "Experience the Magic of Lapland"

**What you can edit:**
- Main title and subtitle
- Feature descriptions (the 4 boxes below the title)
- Button text

**How to edit:**
1. Click **"Hero Section"** in the left sidebar
2. Click **"Hero (Swedish)"** (or other language)
3. Edit the fields
4. Click **"Publish"** to save

### 2. Experiences Section
Information about the different activities (snowmobile tours, Northern Lights, dog sledding, lodging)

**What you can edit:**
- Activity titles and descriptions
- Feature lists for each activity
- Call-to-action text

**How to edit:**
1. Click **"Experiences"** in the left sidebar
2. Click **"Experiences (Swedish)"** (or other language)
3. Edit the fields
4. Click **"Publish"** to save

## Editing Workflow

### Step 1: Select a Section
Click on a section in the left sidebar (Hero Section, Experiences, etc.)

### Step 2: Choose a Language
Click on the language you want to edit (Swedish, English, German, Polish)

### Step 3: Make Changes
Edit the text fields. You can:
- Change titles and descriptions
- Edit feature lists
- Update button text
- Add or remove content

### Step 4: Preview (Optional)
Click the **"Preview"** button to see how your changes look

### Step 5: Publish
Click **"Publish"** to save your changes to the website

**Important**: Your changes go live immediately after publishing!

## Tips for Success

### Writing Good Content

- **Be clear**: Use simple, direct language
- **Be concise**: Keep descriptions short and focused
- **Be consistent**: Use the same style throughout
- **Check spelling**: Proofread before publishing

### Editing Best Practices

- **Save often**: Click Publish frequently
- **One section at a time**: Edit one section, then move to the next
- **Test changes**: View the website after publishing to verify
- **Keep backups**: Important changes are saved in GitHub

### What NOT to Do

- ❌ Don't change field names (only edit values)
- ❌ Don't delete entire sections
- ❌ Don't add new fields without contacting Joakim
- ❌ Don't publish without reviewing

## Common Tasks

### Changing the Hero Title

1. Click **"Hero Section"** → **"Hero (Swedish)"**
2. Find the **"Title"** field
3. Change "Upplev" to your new text
4. Click **"Publish"**

### Updating Experience Descriptions

1. Click **"Experiences"** → **"Experiences (Swedish)"**
2. Find the section you want to edit (e.g., "Snowmobile Title")
3. Edit the text
4. Click **"Publish"**

### Adding Feature Information

1. Click **"Hero Section"** → **"Hero (Swedish)"**
2. Find the feature fields (Feature 1 Title, Feature 1 Desc, etc.)
3. Edit the text
4. Click **"Publish"**

## Troubleshooting

### "Login Failed"

- Make sure you're using the correct GitHub account
- Check that you have access to the repository
- Try logging out and logging back in

### "Publish Failed"

- Wait a few seconds and try again
- Check your internet connection
- Try refreshing the page
- Contact Joakim if the problem persists

### "Changes Not Appearing"

- Wait 1-2 minutes for the website to update
- Refresh your browser (Ctrl+F5 or Cmd+Shift+R)
- Check the website on a different device
- Contact Joakim if changes don't appear

### "I Accidentally Changed Something"

- Don't panic! Changes are saved in GitHub
- Contact Joakim to revert changes if needed
- GitHub keeps a complete history of all changes

## Languages

The CMS supports 4 languages:

- **Swedish (SV)** - Master language (edit this first)
- **English (EN)** - Translated version
- **German (DE)** - Translated version
- **Polish (PL)** - Translated version

**Important**: Edit the Swedish version first. Other languages are translations and should match the Swedish content.

## What Happens When You Publish

1. **You click "Publish"**
2. **CMS saves your changes to GitHub**
3. **Website automatically rebuilds** (takes 1-2 minutes)
4. **Your changes go live**

Everything happens automatically—no need to do anything else!

## Keyboard Shortcuts

- **Tab**: Move to next field
- **Shift+Tab**: Move to previous field
- **Enter**: Save (in some fields)
- **Ctrl+S** / **Cmd+S**: Save (in some browsers)

## Need Help?

If you have questions or run into problems:

1. Check this guide first
2. Try refreshing the page
3. Clear your browser cache
4. Contact Joakim with:
   - What you were trying to do
   - What error you saw (if any)
   - Screenshots if possible
