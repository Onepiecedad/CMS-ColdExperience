# ColdExperience CMS - Editor Guide for Gustav

Welcome to the ColdExperience CMS! This guide will help you manage content and images without needing to touch any code.

## Quick Start

### Accessing the CMS

1. Go to **https://coldexperience.com/admin** (or your site URL + `/admin`)
2. Click **"Login with GitHub"**
3. Authorize the application with your GitHub account
4. You're in! You'll see the CMS dashboard

## Main Tasks

### Editing the Home Page

1. In the CMS dashboard, click **"Home Page"** in the left sidebar
2. You'll see fields for:
   - **Hero Title**: Main headline
   - **Hero Subtitle**: Subheading
   - **Hero Image**: Upload or change the main image
   - **Hero Video**: Upload a video file
   - **CTA Text**: Button text (e.g., "Explore experiences")
   - **CTA Link**: Where the button goes
   - **About Title & Description**: Information section
   - **Features**: Add feature cards with icons

3. Make your changes
4. Click **"Publish"** to save and deploy

### Managing Experiences

Experiences are the cold water activities you offer.

#### Creating a New Experience

1. Click **"Experiences"** in the left sidebar
2. Click **"New Experiences"**
3. Fill in the details:
   - **Title**: Experience name
   - **Slug**: URL-friendly name (e.g., "ice-plunge")
   - **Summary**: Short description (1-2 sentences)
   - **Description**: Detailed description
   - **Price From**: Starting price in SEK
   - **Duration**: How long (e.g., "1 hour")
   - **Season**: When it's available (Winter, Spring, Summer, Fall)
   - **Difficulty Level**: Beginner, Intermediate, or Advanced
   - **Gallery**: Upload multiple images
   - **Featured**: Check if it should appear on homepage

4. Click **"Publish"**

#### Editing an Experience

1. Click **"Experiences"** in the left sidebar
2. Click the experience you want to edit
3. Make your changes
4. Click **"Publish"**

#### Deleting an Experience

1. Click **"Experiences"** in the left sidebar
2. Click the experience you want to delete
3. Click the **"Delete"** button
4. Confirm the deletion

### Managing FAQ

FAQ stands for Frequently Asked Questions.

#### Adding a Question

1. Click **"FAQ"** in the left sidebar
2. Click **"New FAQ"**
3. Fill in:
   - **Question**: The question text
   - **Answer**: The answer (supports formatting)
   - **Category**: General, Safety, Booking, or Health
   - **Order**: Number for sorting (1, 2, 3, etc.)

4. Click **"Publish"**

#### Editing a Question

1. Click **"FAQ"** in the left sidebar
2. Click the question you want to edit
3. Make your changes
4. Click **"Publish"**

## Uploading Images

### Where Images Go

All images are stored in the media library and automatically organized.

### How to Upload

1. When editing any content, look for an **"Image"** field
2. Click the image field
3. Choose **"Upload"**
4. Select your image file from your computer
5. The image is automatically saved

### Image Tips

- Use high-quality images (at least 1200px wide)
- Compress images before uploading (use tools like TinyPNG)
- Use descriptive filenames (e.g., "ice-plunge-hero.jpg")
- Supported formats: JPG, PNG, WebP, GIF

## What Happens When You Publish

When you click **"Publish"**:

1. Your changes are saved to GitHub
2. A translation system automatically translates your Swedish text to:
   - English
   - German
   - Polish
3. The website automatically rebuilds and updates live
4. All changes are live within 2-5 minutes

## Important Notes

### Swedish is the Master Language

- Always edit content in **Swedish first**
- Other languages are automatically translated
- Don't edit English, German, or Polish directly

### Translations

- Translations happen automatically
- Brand terms (like "ColdExperience") are never translated
- If a translation looks wrong, contact Joakim

### Preview Before Publishing

- You can preview changes before publishing
- Look for the **"Preview"** button in the CMS
- Check that everything looks correct
- Then click **"Publish"** to go live

## Troubleshooting

### "Login Failed"

- Make sure you're using the correct GitHub account
- Check that your GitHub account has access to the repository
- Try logging out and logging back in

### "Changes Not Appearing"

- Wait 2-5 minutes for the site to rebuild
- Refresh your browser (Ctrl+F5 or Cmd+Shift+R)
- Check if there are any error messages in the CMS

### "Image Won't Upload"

- Check the file size (should be under 10MB)
- Try a different image format (JPG instead of PNG)
- Check your internet connection
- Contact Joakim if the problem persists

### "Text Looks Wrong"

- Check for typos
- Make sure you're editing in Swedish
- Check that all required fields are filled
- Contact Joakim if translations look incorrect

## Tips for Success

1. **Save Often**: Click "Publish" frequently to save your work
2. **Preview First**: Always preview before publishing
3. **Use Clear Language**: Write in clear, simple Swedish
4. **Check Spelling**: Proofread before publishing
5. **Consistent Formatting**: Use the same style for similar content
6. **Descriptive Images**: Use high-quality, relevant images
7. **Keep It Updated**: Update content regularly to keep the site fresh

## Getting Help

If you have questions or run into problems:

1. Check this guide first
2. Try refreshing the page
3. Clear your browser cache
4. Contact Joakim with:
   - What you were trying to do
   - What error message you saw (if any)
   - Screenshots if possible

## Keyboard Shortcuts

- **Ctrl+S** (Windows) or **Cmd+S** (Mac): Save
- **Ctrl+Z** (Windows) or **Cmd+Z** (Mac): Undo
- **Tab**: Move to next field
- **Shift+Tab**: Move to previous field
